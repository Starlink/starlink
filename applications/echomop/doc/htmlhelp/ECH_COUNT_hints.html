Hints

It is fairly important that the initial counting of the orders yields the correct result as it is used to size the sampling box when doing the order location. Problems may be caused by bad pixels (See TUNE_FCHECK) or by cosmic rays (See ECH_DECOSMIC_1, although tracing should usually be done using a flat-field frame).

In the event that there are still problems, it is possible to manually specify the number of orders present using FIGARO SETOBJ: 

    SETOBJ object=/rdctn-file-name/.MORE.ECHELLE.NO_OF_ORDERS value=/nn/

at the system shell prompt.
