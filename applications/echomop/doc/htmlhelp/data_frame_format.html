Data Frame Format

The input data frames are expected to be in Starlink sdf files. The data themselves being stored in a 2-dimensional array accessible via the path 

    - /filename/.NDF.DATA_ARRAY

In fact, ECHOMOP is more flexible than many Starlink programs and can accept any object in an sdf as a data frame array. Thus an sdf could contain multiple frames for a single observing run: 

    - filename/.QSO.NDF.DATA_ARRAY


    - filename/.QSO2.NDF.DATA_ARRAY


    - filename/.GALAXY.NDF.DATA_ARRAY


    - filename/.FLAT.DATA

This capability may be useful when a foreign data format has been translated into an sdf, but with non-standard object names.
