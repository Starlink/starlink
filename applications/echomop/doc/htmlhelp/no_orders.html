No_orders

The task has been unable to find any orders in the frame supplied.

The most likely cause is that the frame has been incorrectly specified and contains no orders. Another possibility is that the frame has the wrong orientation (orders long axis not horizontal) although at least one order would usually be 'found' in such a case. If the frame is correctly specified and orientated then possible causes are as follows:

  • Cosmic ray noise; use the ech_decos1 option to clean the cosmic rays before re-trying order location.

  • Very poor S/N data; set the parameter TUNE_AUTLOC=Y to specify the positions of the orders manually using a graphics cursor.
