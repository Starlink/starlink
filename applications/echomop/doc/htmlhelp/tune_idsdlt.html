TUNE_IDSDLT

Usage

TUNE_IDSDLT=_INTEGER

Description

Specifies the starting number of neighbouring features (on each side) to examine when generating ratios for pattern matching.

Increasing the value of this parameter will lead to an exponential increase in CPU time.

Higher values are automatically tried by the program if no solution can be found. The number of neighbours considered is incremented until it reaches the maximum specified by TUNE_IDMDLT, at which point the program will give up automatic matching.

The value of TUNE_IDMDLT should only be increased when all other possibilities for wavelength fitting have been exhausted.

The default value is 3.
